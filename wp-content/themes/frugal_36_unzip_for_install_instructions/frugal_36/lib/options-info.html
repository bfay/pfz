About Saving Changes...

A quick note about saving changes in frugal. Because frugal is setup to be cached by browsers you may not see saved changes until you delete your own browsers cache. The easiest way to do this is to just click the 'Refresh' button that is usually located next to your browser's address bar. The reason for all of this is to make your frugal website super fast and efficient. So when a new visitor comes to your site to look around your web server will only have to serve them the bulk of your site information once (at least until they clear their cache or a certain time period has elapsed. This makes it so much of your sites design is loaded into their browser and then accessed locally from their computer and therefore requires little use of your server after that initial visit. So it's all a good thing, but will require the occasional 'refresh' of your browser when making adjustments to your frugal Theme Options.

CSS vs. PHP...

The CSS/PHP option may be a bit confusing so let me explain. The simple answer to the use of this option is that shouldn't touch it unless your style options aren't taking effect and you're unable to change the permissions of the 'dynamic.css' file on your server.

There are two ways for your frugal Wordpress Theme to change its CSS settings. One is to hard code these updated settings to a CSS file called 'dynamic.css'. This is nice because it provides the web browsers of the people viewing your site the ability to completely cache your styles. This provides super fast and super efficient use of your site.

The other way that frugal updates these options is to do so reading directly from the PHP file that computes these dynamic styles. Because these styles are all initiated and read with a PHP file there is no need for the file to be writable. So this is the option that will assure that your settings will change when you click that big, fat SAVE button (or the little guy I added to the bottom of your options pages). But because these styles are dynamically adjusted they are a bit less efficient than the CSS option, but only the slightest bit as this option is also cache-able by browsers, but in a different way. So either way you're going to have a fast, efficient website when frugal's at the helm. :-)

Margin And Padding Options...

Just in case you didn't know what areas pertain to what (eg. 10px 10px 10px 10px) as far as which number changed what setting (eg. top - right - bottom - left). It actually goes in that order. So the first px'd number pertains to the top, the second to the right, the third to the bottom and the fourth to the left. So it goes clockwise, starting from the top.

Custom Files...

You'll also find two custom files that will help you if you're still in need of greater customization capability. First, you'll find a 'cutom.css' file located in the root directory of your frugal Wordpress Theme. You can easily add to and edit this file via the Wordpress 'Editor' under the 'Appearance' section in your Wordpress Dashboard. By adding custom styles to this file you can override ALL other CSS settings in your theme. The other custom file is a 'custom_functions.php' file that is located in the /custom/ directory. You can add custom functions to this file as the need arises.

The Master Reset Switch...

One of these options I want to explain further is the 'Master Reset Switch'. What this does is completely reset all your frugal theme options to their stock settings. I've included this option for two reasons. One, it can be a great way to 'fix' a broken theme. If for whatever reason your new settings conflict with those of a previous installation you can reset the theme to clean up your server database which usual solves most theme issues. Just keep in mind that you are WIPING your settings, so you'll have to re-add any changes you've made.

The second use for this option is just being able to start from scratch when working on a new design. Kind of like balling up your paper and tossing it so you can grab a fresh sheet, this option allows you to wipe the slate clean so you begin the design process from the beginning.
